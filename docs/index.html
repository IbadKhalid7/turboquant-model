Paper → Practice: What This Project Does

This project takes the core ideas from the TurboQuant paper and applies them to offline LLM weight compression with fused inference kernels — a different use case than the paper's primary focus.

📄 The Paper (Zandieh et al., 2025)

The TurboQuant paper introduces a framework for online vector quantization with near-optimal distortion rate. Its primary contribution is TurboQuantprod — an unbiased inner product estimator that combines Lloyd-Max scalar quantization with a 1-bit QJL (Quantized Johnson-Lindenstrauss) correction for applications like KV-cache attention, where you quantize keys once and query with many different vectors.

Online estimation QJL 1-bit correction Unbiased dot products KV-cache focus

⚡ This Project

We take the paper's rotation + Lloyd-Max quantization foundation and apply it to offline weight compression — quantize each nn.Linear once, then serve with on-the-fly dequantization. We deliberately replace QJL with multi-pass residual quantization using full Lloyd-Max codebooks, and add fused GPU kernels for production inference.

Offline weight compression No QJL Residual quantization Fused CuTile/Triton kernels

Key Differences from the Paper

🚫

No QJL Correction

QJL's 1-bit unbiased estimator is designed for streaming dot products (KV-cache). For weight compression, a deterministic bias from optimal quantization is absorbed by layer norms — an unbiased but high-variance 1-bit correction hurts more than it helps.

🎯

Residual Quantization Instead

Instead of a 1-bit QJL pass, we run a full second TurboQuant pass on the residual error with its own Lloyd-Max codebook. 4+4 residual achieves KLD 0.002 (near-lossless) — strictly dominating QJL's 1-bit correction.

🔄

Rotate Input, Not Weight

At inference time, we pre-rotate the activation (B×d, cheap) instead of inverse-rotating the full weight matrix (M×N, expensive). Weights stay packed as 4-bit indices — never materialized.

🚀

Fused GPU Kernels

CuTile and Triton kernels fuse unpack→lookup→matmul→rescale into a single launch. The 64-byte codebook lives in registers. Up to 4× speedup and 5.7× memory reduction vs naive PyTorch.

Core Techniques

TurboQuant combines six key techniques to achieve near-information-theoretic-optimal weight compression.

📊

Lloyd-Max Scalar Quantization

Optimal quantizer for Gaussian distributions. Iteratively refines 16 centroids and decision boundaries to minimize MSE. Computed once for 𝒩(0,1) — just 64 bytes shared across all layers.

🔄

Random Orthogonal Rotation

Multiplying by a random orthogonal matrix Π spreads information uniformly across all coordinates, decorrelating weights and Gaussianizing them via the central limit theorem.

Π
⚡

Walsh-Hadamard Transform

O(d log d) butterfly-style rotation via the fast Walsh-Hadamard transform + random sign flips. O(d) storage vs O(d²) for QR — faster and leaner with identical quality.

🎯

Residual Quantization

Two-pass quantization: Pass 1 captures coarse structure, Pass 2 captures the residual error. 4+4 bits achieves KL divergence of only 0.002 nats — practically lossless.

📦

4-bit Index Packing

Two 4-bit indices packed per uint8 byte — halving storage. Bitwise operations in registers make packing/unpacking nearly free.

🚀

Fused GPU Kernels

CuTile and Triton kernels fuse unpack → lookup → matmul → rescale into a single launch. The 64-byte codebook lives in registers — zero global memory round-trips.

Quick Start

Get running in minutes. Drop-in replacement for nn.Linear — no model architecture changes.

Installation

# Install with transformers support
uv pip install -e ".[transformers]"

Python API

from transformers import AutoModelForCausalLM, AutoTokenizer
from turboquant_model import TurboQuantConfig, quantize_model

# Load model
model = AutoModelForCausalLM.from_pretrained("Qwen/Qwen3.5-0.8B-Base").cuda()

# Quantize — single-pass 4-bit
config = TurboQuantConfig(bit_width=4, seed=42)
model = quantize_model(model, config)

# Residual quantization — near-lossless 4+4
config = TurboQuantConfig(bit_width=4, residual_bit_width=4, seed=42)
model = quantize_model(model, config)

CLI

# Quantize and save
turboquant quantize --model Qwen/Qwen3.5-0.8B-Base --output ./quantized --bit-width 4

# 4+4 residual (near-lossless)
turboquant quantize --model Qwen/Qwen3.5-0.8B-Base --output ./quantized \
  --bit-width 4 --residual-bit-width 4

# Evaluate perplexity
turboquant eval --model Qwen/Qwen3.5-0.8B-Base --quantized ./quantized

# Generate text
turboquant generate --model Qwen/Qwen3.5-0.8B-Base --quantized ./quantized \
  --prompt "The capital of France is"

Architecture

turboquant_model/
├── codebook.py       # Lloyd-Max optimal codebook (precomputed)
├── rotation.py       # Random rotation: QR (Haar) or fast Walsh-Hadamard
├── quantize.py       # Single-pass quantize + pack/unpack
├── residual.py       # Residual (multi-pass) quantization
├── cutile_kernels.py # Fused CuTile matmul kernel
├── triton_kernels.py # Fused Triton matmul kernel
├── module.py         # TurboQuantLinear (nn.Module)
├── model.py          # quantize_model, save/load, config
└── cli.py            # Command-line interface
